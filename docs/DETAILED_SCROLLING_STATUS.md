# Detailed Scrolling Processing Status — v1.5.7

This version expands the scrolling processing status feed.

It now shows:

- frontend status lines before and during backend call
- segmentation mode/quality for each view
- every gate result as a separate status line
- every individual parameter comparison as a separate PASS/FAIL line
- no-match reasons
- final decision

The purpose is to make the processing transparent so users can follow every check performed by the engine.
